Installing and Setting Up SQLite

Installing and setting up MySQL for use with Shoko Server.

Why SQLite

SQLite is a lightweight database engine that is portable and free. It requires no database configuration and Shoko handles the need to run a seperate database server.

While SQLite is the easiest of the three database types to setup with it’s one-click setup. If your collection contains 200 or more series, you will start to experience a lot of issues including slow response times and program lockups when Shoko attempts to access your collection via your SQLite database. In regards to how Shoko uses the database, it’s simply not designed for large collections. SQLite also has difficulty with handling multiple operations which increases the wait time between operations which overall increase your wait time when accessing your collection. It is therefore recommended that you consider using another database option when dealing with a collection that contains 200 or more series.

SQLite Installation

No requirements are needed to perform a SQLite Installation. If you believe your collection will contain less than 200 series, you may continue with the installation of Shoko Server.

If you have encountered an issue and wish to reset your SQLite database, then it is as simple as deleting the JMMServer.db3 file.

Default Setting Options Notes
"C:\ProgramData\ShokoServer\SQLite\JMMServer.db3" N/A Default location for SQLite Database.
